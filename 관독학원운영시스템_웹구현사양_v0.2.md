## 0. 핵심 원칙 (v0.2 §1 정합)

> **시스템이 만들고, 매니저는 검토·처방·면담만 한다.** LETS·에듀플렉스가 다루는 학원 운영 영역을 **하나도 빼지 않고** 다루되, 매니저의 "만드는 부담"을 0에 가깝게.

## 1. 매니저 업무 매트릭스

| 업무 영역 | 시스템이 흡수 | 매니저가 직접 | 학생당 빈도 |
|---|---|---|---|
| 학습 코칭·출결 | 자동 체크·시간표·외출/졸음 알림 | 결석 학부모 컨택, 막힌 학생 안내 | 매일 ~1분 |
| 일일 학습 리포트 | **자동 초안 생성** | 한 줄 코멘트, 발송 컨펌 | 매일 ~1분 |
| 모의고사 자료 | 약점별 **자동 추천** | 추천 컨펌, 출력·전달 | 시험 전 ~5분 |
| 내신 문제 | 학교·단원별 **자동 매칭** | 추천 컨펌, 변형 1~2개 | 시험 전 ~5분 |
| 과목별 질의응답 | 콘텐츠 모듈 + 자동 퀴즈 | 막힌 부분 1:1 보충 | 알림 시만 |
| 학생부 특기사항 | 데이터에서 **자동 추출** | 학기말 다듬기 | 학기 1~2회 |
| 수행평가 관리 | 일정 알림·체크리스트·자료 매칭 | 결과물 코멘트 | 주 ~5분 |
| 시험 분석 | 점수 → **자동 분석 리포트** | 분석 검토, 면담 자료 | 시험 후 ~10분 |
| 입시 컨설팅 | 학생 데이터 풀 자동 정리 | 1:1 면담 진행 | 학기 1~2회 |
| 과목 특화 (수학 등) | 진도·수준별 **자동 추천** | 진도 점검·결정 | 주 ~5분 |
| 위기 시그널 대응 | 패턴 **자동 감지·알림** | 1:1 개입 | 알림 시만 |

### 학생 30명 기준 매니저 주간 부담

| 항목 | 시간 |
|---|---|
| 출결·일일 코멘트 (평일 30분/일 × 5) | 2.5h |
| 학부모 리포트 컨펌 (주 30건) | 0.5h |
| 콘텐츠·수행평가 점검 (주 30명 × 5분) | 2.5h |
| 위기 시그널 대응 | 1~2h |
| **일반 주 합계** | **~7시간** |
| 시험기 추가 (월 1~2회) | 주 7.5h 추가 |

> **결과**: 일반 주 매니저 1명 = 파트타임으로 20~30명. 에듀플렉스 8~12명 대비 2~3배.

## 3. 학생 포털 화면 사양

### 3.0 디바이스 정책 (신규)

학생은 **PC와 스마트폰 둘 다** 사용. **반응형 웹**으로 구현 (별도 앱 없음).

| 디바이스 | 위치 | 주 사용 화면 | 주 작업 |
|---|---|---|---|
| PC (학원 내) | 등원 후 메인 | 홈, D-21 그리드, 콘텐츠 학습, 일일 입력 | 깊은 작업 (영상 학습, 그리드 채우기, 회고 작성) |
| 스마트폰 (외부) | 학원 밖 | 홈 요약, 회독 체크, 컨디션 입력, 콘텐츠 영상 | 빠른 입력 (회독 체크, 진척 확인, 영상 시청) |

- 데이터 양쪽 **실시간 동기화** (학원에서 PC로 작성한 D-21 그리드를 모바일에서 확인·수정)
- 매니저 화면도 반응형이지만 PC 우선 (모바일은 알림 확인용)

### 3.1 홈 (Today) — 매일 등원 시 첫 화면
- 오늘 날짜·시즌 단계 (Build/Race/Battle/Autopsy)
- 다음 시험까지 D-Day
- 오늘 할 일 3~5개 (시즌 단계 기반 자동 생성)
- 추천 콘텐츠 모듈 1개
- 빠른 입력 위젯 (어제 진척 확인)

### 3.2 D-21 역산 그리드 (LETS 이미지 3 정조준)
- 시험일 + 과목 입력 시 자동 21일 격자 생성
- 각 칸: 그 날 회독 단계·분량·콘텐츠 모듈
- 매일 체크 → 진척률 자동 계산
- "필수 외 버려야 할 활동" 입력 (LETS 이미지 2의 뺄셈 전략)
- 매니저 코멘트 표시 (있을 때만)

### 3.3 N회독 트래커 (LETS 이미지 4 정조준)
- 과목별 핵심 자료 등록 (교과서·프린트·문제집)
- 1·2·3회독 체크
- D-14·D-7·D-1 자동 목표일
- 단권화 점검 체크리스트
- 회독 단계 → 시스템이 자동 안내 ("이 단계엔 이 문제집 풀어")

### 3.4 콘텐츠 학습 (변경: 자체 호스팅)

- **자체 호스팅. YouTube 사용 안 함.**
- C1~C8 영상 라이브러리 (시즌 단계별 추천)
- 영상 + 워크북 + 진척 체크
- 모듈 완료 = 시스템 진척 반영

**호스팅 옵션 트레이드오프**:

| 서비스 | 비용 (월) | 도메인 제한 | DRM | 기술 부담 |
|---|---|---|---|---|
| Vimeo Pro | ~$20 | ✓ | 약 | 낮음 (즉시 시작) |
| Vimeo Plus + 도메인 잠금 | ~$7 | ✓ | 약 | 낮음 |
| Cloudflare Stream | 분당 과금 (~$5/1000분) | ✓ | 강 | 중 |
| AWS S3 + CloudFront + signed URL | 사용량 기반 | ✓ | 강 | 높음 |

**Phase 1 권장**: Vimeo Pro 시작 (즉시 시작 + 기능 충분). 스케일 시 Cloudflare Stream 전환 검토.

### 3.5 일일 입력 — 매일 ~3분
- 오늘 한 분량 / 못 한 분량 / 사유
- 컨디션 (수면·집중도) 1~5점
- 자유 메모 (선택)

### 3.6 시험 회고 (Autopsy)
- 시험 후 7일 자동 활성화
- 오답 원인 분류 (실수 vs 몰라서 vs 시간 부족)
- 다음 시즌으로 약점 자동 이월

### 3.7 내 누적 데이터
- 이전 시즌 진척 그래프
- 회독 완수 패턴
- 컨디션 트렌드
- 자동 코칭 멘트 ("지난 시험에 너 D-7에 N회독 못 끝냈잖아")

## 4. 매니저 포털 화면 사양

### 4.1 대시보드 (메인)
*매니저 대시보드 와이어프레임 참조. 다음 단계에서 기존 AM 대시보드 자산과 매핑하여 정밀화.*
- 위기 시그널 알림 (최상단)
- 오늘 할 일 (학부모 리포트 컨펌·일일 코멘트·OJT·시험 분석·수행평가·맞춤 처방)
- 담당 학생 카드 그리드 (상태별 색상)
- 이번 주 통계

### 4.2 학생별 상세
- 학생 1명 클릭 → 모든 데이터 한 화면
- 시즌 진행 / D-21 / N회독 / 콘텐츠 진척
- 시험 분석 히스토리
- 학부모 리포트 발송 이력
- 수행평가 일정·상태
- 매니저 메모

### 4.3 일일 학습 리포트
- 담당 학생 N명 자동 초안 생성
- 매니저는 학생당 한 줄 코멘트만
- "전체 일괄 발송" 버튼 (학부모 리포트 연동)

### 4.4 학부모 리포트 컨펌
- 주 1회 자동 생성된 리포트 미리보기
- 학생당 1~3분 검토
- 컨펌 → 카톡 알림톡 자동 발송 (§6.1 연동)

### 4.5 문제 은행 (변경: 하이브리드형)

**하이브리드 구조**:
- **본문**: PDF 첨부 (학원 기존 자료 즉시 활용)
- **메타데이터** (DB 입력 필수):
  - 학교 / 학년 / 학기
  - 과목 / 단원 (대단원·중단원)
  - 유형 (개념·적용·서술형·실수형·고난도·빈출·오답 다발)
  - 난이도 (1~5)
  - 기출년도 / 출처 (학교 기출·모의고사·시중 문제집)
  - 정답률 (학원 누적 데이터)
- **검색·추천**: 메타데이터 기반
  - 예: "OO고 2학년 2학기 기말 빈출 + 학생 약점(이차함수) + 난이도 3" 자동 매칭
- **Phase 2+**: 자주 쓰는 문제 → 텍스트 DB 전환 (변형 출제 자동화)

> **결정 필요 (신규)**: 메타데이터 분류 체계 (단원 명명 규칙·유형 분류 등) 정의 별도 작업.

### 4.6 학생별 맞춤 문제 처방
- 시스템 추천 → 매니저 컨펌 → 학생에게 자동 배포
- 학생 풀이 결과 → 다음 처방에 반영

### 4.7 수행평가 관리 (**Phase 1 끌어올림**)
- 학생별 수행평가 일정 캘린더
- 시스템: 일정 7일 전 알림, 필요 자료 자동 매칭
- 매니저: 학생 결과물 코멘트
- Phase 1엔 간략 버전 (일정·체크·코멘트). 자료 자동 매칭은 Phase 2.

### 4.8 시험 분석 (**Phase 1 끌어올림**)
- 점수 입력 → 자동 분석 (취약 단원·취약 유형·실수 vs 몰라서)
- 학부모 면담 자료 자동 생성
- 다음 시즌 약점으로 자동 이월
- Phase 1엔 간략 버전 (점수 입력·기본 분석). 면담 자료 자동 생성은 Phase 2.

## 6. 학부모 채널 (변경: 카톡 비즈메시지)

**학부모 포털 없음**. 카톡 비즈메시지 알림톡 + (선택) 토큰 기반 열람 페이지.

### 6.1 카톡 알림톡 발송 시스템

- **카카오 비즈메시지 알림톡 채택**
- 비용: 메시지당 ~9원 (알림톡), ~3원 (친구톡)
- **학원당 월 비용**: 학생 30명 × 주 1회 × 4주 = 120건 × 9원 = **~1,080원/월**
- **사전 작업** (Phase 1 빌드 전):
  1. 카카오 비즈니스 계정 생성 (사업자등록증 필요)
  2. 알림톡 채널 등록 (학원 채널)
  3. 알림톡 템플릿 사전 등록·승인 (~1주 소요)
  4. 학생 등록 시 학부모 카톡 ID 수집 (등원 폼에 추가)
- **SMS 폴백**: 알림톡 실패 시 자동 SMS 전환

**알림톡 템플릿 4종 (사전 승인 필요)**:
1. 주간 자동 리포트
2. 시즌 종합 리포트 (시험 후)
3. 결석 알림
4. 위기 시그널 알림

> **결정 필요 (신규)**: 알림톡 4종 템플릿 본문 작성 (정연 톤 반영) — 별도 작업.

### 6.2 (선택) 학부모 열람 페이지
- 토큰 기반 1회용 URL (로그인 X)
- 시즌 종합 리포트 / 누적 그래프 열람 전용
- Phase 2+에서 검토
